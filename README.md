#### Техническое задание к программе "Калькулятор".

Цель: реализовать программу "Калькулятор" на языке программирования C# (.Net) с консольным интерфейсом взаимодействия с пользователем, выполняющую ряд математических операций над двумя, вводимыми пользователем, числами.

###### Функционал для реализации:

1. выполнение математической операции сложения двух заданных чисел;
2. выполнение математической операции вычитания двух заданных чисел;
3. выполнение математической операции умножения двух заданных чисел;
4. выполнение математической операции деления двух заданных чисел;
5. выполнение математической операции возведения в степень.

###### Алгоритм взаимодействия программы с пользователем:

1. пользователь запускает программу;
2. программа отображает пользователю инструкцию для ввода данных;
3. программа приглашает пользователя ко вводу данных по инструкции;
4. пользователь вводит математическое выражение, состоящее из двух произвольных чисел и допустимой операции, в формате, описанном в инструкции;
5. программа производит вычисления;
6. программа выводит результат вычисления или сообщение об ошибке, возникшей в процессе вычисления или обработки ввода;
7. программа должна завершаться при вводе указанного в инструкции значения;
8. программа продолжает свою работу с пункта 3.

###### Требования к программной реализации:

1. программа должна быть реализована на языке программирования C# (12-й или старшей версии), на платформе .NET;
2. программа должна взаимодействовать с пользователем через консольный интерфейс;
3. программа должна выводит пользователю инструкции о работе с ней;
4. программа должна приглашать пользователя ко вводу и заранее описывать формат ввода;
5. все инструкции и информация о работе программы на этапе взаимодействия с пользователем должны быть на русском языке;
6. программа должна продолжать своё выполнение при возникших ошибках и уведомлять о них пользователя;
7. программа должна уведомлять пользователя о некорректности ввода;
8. программа должна выполнять математические операции сложения, вычитания, умножения, деления и возведения в степень рациональных чисел;
9. программа должна быть простой в расширении функционала.

###### Тест-кейсы, определяющие требования к логике работы программы:

1. тест-кейсы, определяющие требования к функционалу сложения двух чисел:

   * ввод: "4 + 2", ожидаемый вывод: "Результат вычисления: 6";
   * ввод: "-4,2 + 2,2", ожидаемый вывод: "Результат вычисления: -2";
   * ввод: "9999999999999999999999999999999999999999999999999999999999999999999999999999999 + 999999999999999999999999999999999", ожидаемый вывод: "Ошибка: введённое число слишком большое";
   * ввод: "abc + def", ожидаемый вывод: "Ошибка: некорректный ввод";

2. тест-кейсы, определяющие требования к функционалу вычитания двух чисел:

   * ввод: "4 - 2", ожидаемый вывод: "Результат вычисления: 2";
   * ввод: "-4,2 - 2,2", ожидаемый вывод: "Результат вычисления: -6,4";
   * ввод: "9999999999999999999999999999999999999999999999999999999999999999999999999999999 - 999999999999999999999999999999999", ожидаемый вывод: "Ошибка: введённое число слишком большое";
   * ввод: "abc - def", ожидаемый вывод: "Ошибка: некорректный ввод";

3. тест-кейсы, определяющие требования к функционалу умножения двух чисел:

   * ввод: "4 \* 2", ожидаемый вывод: "Результат вычисления: 8";
   * ввод: "-4,2 \* 2", ожидаемый вывод: "Результат вычисления: -8,4";
   * ввод: "9999999999999999999999999999999999999999999999999999999999999999999999999999999 \* 999999999999999999999999999999999", ожидаемый вывод: "Ошибка: введённое число слишком большое";
   * ввод: "abc \* def", ожидаемый вывод: "Ошибка: некорректный ввод";

4. тест-кейсы, определяющие требования к функционалу деления двух чисел:

   * ввод: "40 / 2", ожидаемый вывод: "Результат вычисления: 20";
   * ввод: "-4,2 / 2", ожидаемый вывод: "Результат вычисления: -2,1";
   * ввод: "9999999999999999999999999999999999999999999999999999999999999999999999999999999 / 999999999999999999999999999999999", ожидаемый вывод: "Ошибка: введённое число слишком большое";
   * ввод: "abc / def", ожидаемый вывод: "Ошибка: некорректный ввод";
   * ввод: "4 / 0", ожидаемый вывод: "Ошибка: недопустимая операции";

5. тест-кейсы, определяющие требования к функционалу возведения в степень:

   * ввод: "4 ^ 2", ожидаемый вывод: "Результат вычисления: 16";
   * ввод: "4 - 0,5", ожидаемый вывод: "Результат вычисления: 2";
   * ввод: "9999999999999999999999999999999999999999999999999999999999999999999999999999999 ^ 999999999999999999999999999999999", ожидаемый вывод: "Ошибка: введённое число слишком большое";
   * ввод: "abc ^ def", ожидаемый вывод: "Ошибка: некорректный ввод";

6. тест-кейсы, обозначающие реакцию программы на некорректный формат ввода:

   * ввод: "2 + ", ожидаемый вывод: "Ошибка: некорректный ввод"
   * ввод: " \* 7", ожидаемый вывод: "Ошибка: некорректный ввод"
   * ввод: "2 $ 7", ожидаемый вывод: "Ошибка: операция '$' не содержится в системе"
